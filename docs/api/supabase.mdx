Fold Health uses **Supabase** as its backend, providing PostgreSQL database, authentication, real-time subscriptions, file storage, and edge functions.

## Client setup

```
src/lib/supabase.js
```

The client is initialized with environment variables (or hardcoded fallbacks for development):

| Variable | Purpose |
|----------|---------|
| `VITE_SUPABASE_URL` | Supabase project URL |
| `VITE_SUPABASE_ANON_KEY` | Public anon key (RLS-protected) |

A single `supabase` instance is exported and used throughout the app.

## Data mappers

Mappers in `src/lib/` translate between database naming (snake_case) and JavaScript (camelCase).

| Mapper | File | Tables |
|--------|------|--------|
| Patient | `patientMapper.js` | patients, all_patients |
| Call Details | `callDetailsMapper.js` | call_details |
| Analytics | `analyticsMapper.js` | analytics_* tables |
| Embedded | `embedMapper.js` | embed_domains, embed_components |

Each exports `dbToJs(row)` and `jsToDb(obj)`.

## Auto-seeding

`src/lib/seedDatabase.js` runs once per session on app load. It checks if key tables are empty and populates them with fallback data using `upsert` for idempotency.
